## 零、设计摘要

当前两个缺口，一起堵：

1. **工具永不损坏**：一把石斧砍一万棵树。制作系统做完一次就失去意义——经济循环断了。
2. **殖民者只会发呆**：他们有饥饿口渴、会闲逛，但不会帮你干活。基地不会自己运转。

这两个是同一个硬币的两面：**工具会坏（消耗侧）+ 殖民者可以代劳（生产侧）**。一起做——物品有寿命 → 制作有持续需求 → 殖民者自动填补供给 → 殖民地开始自己呼吸。

**核心借鉴**：
- **CDDA 叠堆磨损**：工具不单独升格为 Entity，用 count 存耐久，磨损到零消失
- **DF 优先级链**：殖民者按固定顺序找活干，不需要玩家指派
- **RimWorld 隐式存储**：搬运目标 = 营火周围，不需要划定存储区

**零 UI 新增**。不设计工作指派面板、不搞 Manager 弹窗、不加 ECS 组件。

## 一、工具损耗

### 1.1 核心规则

| 规则 | 实现 |
|------|------|
| 工具不叠加 | `Pile::add()` 遇到工具→永远新建 slot，不和同种合并。每手至多一个工具 |
| 耐久 = count | `(StoneKnife, 300)` = 一把能用 300 次的新刀。旧语义 `(StoneKnife, 5)` = 5 把新刀 |
| 磨损递减 | 每次使用 `count -= wear`（掷骰范围） |
| 耐久耗完消失 | `count <= 0` → 从手/Pile 移除 |
| 材料不变 | Wood / Berry / Stick 等照常叠加 |

### 1.2 如何区分工具 vs 材料

`items.ron` 加 `stackable` 字段——不声明的默认 `true`。声明 `stackable = false` 的物品不参与合并。

```ron
// 工具
ItemDef(key: "stone_axe",     stackable: false, max_durability: Some(500),  wear_per_use: Some((3, 5)))
ItemDef(key: "stone_hammer",  stackable: false, max_durability: Some(400),  wear_per_use: Some((3, 5)))
ItemDef(key: "stone_knife",   stackable: false, max_durability: Some(300),  wear_per_use: Some((2, 4)))
ItemDef(key: "bone_knife",    stackable: false, max_durability: Some(100),  wear_per_use: Some((4, 8)))
ItemDef(key: "wood_knife",    stackable: false, max_durability: Some(50),   wear_per_use: Some((6, 12)))
ItemDef(key: "sharp_stick",   stackable: false, max_durability: Some(120),  wear_per_use: Some((5, 10)))
ItemDef(key: "spear",         stackable: false, max_durability: Some(150),  wear_per_use: Some((4, 8)))
ItemDef(key: "torch",         stackable: false, max_durability: Some(1200), wear_per_use: Some((1, 1)))
ItemDef(key: "stone_shovel",  stackable: false, max_durability: Some(350),  wear_per_use: Some((2, 5)))
ItemDef(key: "stone_drill",   stackable: false, max_durability: Some(250),  wear_per_use: Some((3, 6)))

// 材料
ItemDef(key: "wood",          stackable: true)   // ← 默认，可以不写
ItemDef(key: "berry",         stackable: true)
```

### 1.3 Pile::add 改法

```rust
pub fn add(&mut self, item: ItemKind, count: u32) -> bool {
    if count == 0 { return true; }

    let stackable = crate::data::item_def(item.key()).stackable;

    if stackable {
        // 材料：合并同种
        if let Some(slot) = self.slots.iter_mut().find(|s| s.item == item) {
            slot.count = slot.count.saturating_add(count);
            return true;
        }
    }
    // 工具（或新材料）：永远开新 slot
    if self.slots.len() >= MAX_PILE_SLOTS { return false; }
    self.slots.push(PileSlot { item, count });
    true
}
```

### 1.4 Hands::can_take 改法

```rust
pub fn can_take(&self, item: ItemKind) -> bool {
    let stackable = crate::data::item_def(item.key()).stackable;
    if !stackable {
        // 工具：每手至多一把
        return self.left.is_none() || self.right.is_none();
    }
    // 材料：空手或同种可叠
    match (self.left, self.right) {
        (None, _) | (_, None) => true,
        (Some((l, _)), Some((r, _))) => l == item || r == item,
    }
}
```

### 1.5 磨损调用

每个使用工具的动作，在操作成功后调一行：

```rust
// harvest.rs — 砍树
fn apply_wear(app: &mut App, entity: Entity, wear_range: (u32, u32)) {
    let Ok(mut hands) = app.world.get::<&mut Hands>(entity) else { return };
    // 右手优先
    let slot = hands.right.as_mut().or(hands.left.as_mut());
    let Some((kind, count)) = slot else { return };

    let def = crate::data::item_def(kind.key());
    let Some(wear) = def.wear_per_use else { return };
    let damage = rng.gen_range(wear.0..=wear.1);
    *count = count.saturating_sub(damage);

    if *count == 0 {
        app.push_log(format!("{}坏了！", kind.label()));
        *slot = None;
    }
}
```

调用处（harvest/combat/crafting）各自传入对应的 `wear_range`。矛戳狼在 combat.rs，石斧砍树在 harvest.rs，石刀切割在 crafting.rs。

### 1.6 侧边栏显示

```
之前：石斧 ×5
之后：石斧 247/500
```

格式统一：`{label} {current}/{max}`。材料照旧 `×count`。

## 二、殖民者 AI

### 2.1 优先级链

每 tick，殖民者从高到低检查条件，命中则执行。执行为持续操作——砍树不会只砍一下就停，会锁到树倒或工具坏。

```
优先级 1 — 自救（生死关头）
  ├─ 饥饿 < 30 → 去营火 5 格内找熟肉（Pile 搜索）→ 捡起来吃
  ├─ 口渴 < 30 → 去最近的水源 → 喝水
  ├─ 体温 < 20 → 朝最近火源移动，站住回暖
  └─ HP < 30 → 回营地找床睡觉（回复精力=回血）

优先级 2 — 屠宰（肉要烂了！）
  └─ 视野 15 格内有尸体 → 走过去屠宰（Perishable spoilage 越小优先级越高）

优先级 3 — 添柴（篝火是命）
  └─ 营火邻格木头 < 2 根 → 从周围 10 格 Pile 取木头 → 丢到篝火邻格

优先级 4 — 采摘
  └─ 视野 15 格内有 Fruiting 灌木 → 走过去采

优先级 5 — 砍树
  ├─ 手上有石斧 → 视野 20 格内找树 → 走过去砍
  └─ 手上没有 → 搜索周围 15 格 Pile → 有石斧 → 捡 → 砍

优先级 6 — 挖矿
  ├─ 手上有石锤 → 视野 20 格内找岩石 → 走过去挖
  └─ 手上没有 → 同上搜索捡石锤

优先级 7 — 搬运
  └─ 地上散落物品距营火 > 5 格 → 捡起来 → 搬到营火 3×3 区域空地丢掉

优先级 8 — 晃荡
  └─ 随机走一步（距离 ≤ 3，不远离营火 > 25 格）
```

### 2.4 行动锁定

```rust

//   - 当前 action_lock 的目标还在吗？（树没被砍倒/矿没被挖完/尸体没烂完）
//   - 手上有工具吗？（斧子可能坏了）
```

### 2.5 不主动制作

v1 殖民者不自己搓工具。理由：
- 配方需要工具 + 材料 + 篝火邻格 + 足够光照——条件判断链太长
- 材料消耗算谁的？从哪个 Pile 取？做完放哪？
- 这些决策在 v1 不值得建一个完整系统

v1 只做到"捡已有工具用"。玩家负责做好工具丢在营火旁，殖民者负责用。

## 三、两个系统的交叉点

```
玩家砍树 → 石斧 count 500→497→...→15→0（坏了）
  ↓
玩家回营地 → 殖民者之前捡石斧砍的树掉了一地木头
  ↓
玩家 R 打开制作 → 材料够 → 打了三把新石斧 → 丢地上
  ↓
殖民者路过 → 捡起新石斧 → 继续砍树
  ↓
循环成立
```

> 不需要 UI，不需要指派。营地是一圈隐式的供需环——工具坏了有人做（玩家），做好的工具有人用（殖民者），资源有人搬（殖民者）。

## 七、关键决策记录

1. **工具不升格为 Entity**：count = durability 方案是 CDDA 叠堆磨损的变体。方案 B（entity-based）改动面 12+ 文件，风险过高。
2. **零 UI 新增**：殖民者行为纯 AI 驱动，不需要玩家指派。这是故意的——先让"殖民地自己会转"成立，再考虑要不要加控制面板。
3. **搬运目标是营火而不是存储建筑**：隐式规则够用。有储物建筑后再切换到 ContainerTag 识别。
4. **殖民者不做制作**：配方搜索、材料消耗归属、光照判定——这些决策链在 v1 不值得建。v2 再做。
5. **不新增 ECS 组件**：耐久存在 `PileSlot.count` 和 `Hands` 的 `count` 里。语义复用，不引入新类型。
