## 零、设计摘要

六种天气状态：晴 → 阴 → 毛毛雨 → 中雨 → 暴雨 → 雷阵雨。全图统一天气，开局随机。

潮湿 Wet 组件：0=干爽 100=浸透。淋雨涨、烤火降、室内干。速率按 per-tick 写死，不管速度档。

视觉：调色板叠天气乘数 × 日夜乘数；粒子在视口内随机洒；雷阵雨闪白屏。

## 一、天气状态机

### 1.1 六种状态

| 天气 | duration (游戏分钟) | ticks | 调色板 | 粒子 |
|------|:---:|:---:|------|------|
| Clear | 30-90 | 250-750 | 正常 | 无 |
| Overcast | 15-45 | 125-375 | 灰 | 无 |
| Drizzle | 15-40 | 125-333 | 微灰 | '·' Gray, 1/80 |
| Rain | 10-30 | 83-250 | 蓝灰 | '·' Blue, 1/40 |
| Heavy | 5-20 | 42-167 | 暗蓝 | '│' B/W, 1/15 |
| Thunder | 3-10 | 25-83 | 黑蓝 | '│' White 1/10 + 白闪 |

开局随机：`rand::thread_rng().gen_range(0..6)` 从六种里抽一个。

### 1.2 转移矩阵（duration 到期后掷骰）

```
Clear:    →Overcast 60%   →Drizzle 15%    →保持 25%
Overcast: →Drizzle 50%    →Clear 30%      →保持 20%
Drizzle:  →Rain 40%       →Overcast 35%   →保持 25%
Rain:     →Heavy 30%      →Thunder 20%    →Drizzle 30%   →保持 20%
Heavy:    →Thunder 35%    →Rain 40%       →保持 25%
Thunder:  →Rain 50%       →Heavy 30%      →Drizzle 20%
```

### 1.3 App 字段

```rust
pub weather: Weather,
pub weather_timer: u64,  // 当前天气剩余 tick，到期掷骰转移
```

### 1.4 天气数据

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum Weather {
    Clear,
    Overcast,
    Drizzle,
    Rain,
    Heavy,
    Thunder,
}

impl Weather {
    fn wet_rate(self) -> f32;              // 潮湿增长速率 (/tick)
    fn visibility_multiplier(self) -> f32; // 视野乘数
    fn mood_penalty(self) -> f32;          // 直接心情惩罚
    fn fire_extinguish_chance(self) -> f32;// 户外火源熄灭概率
    fn min_duration(self) -> u64;          // ticks
    fn max_duration(self) -> u64;
    fn next(self, rng: &mut impl Rng) -> Self;
    fn label(self) -> &'static str;
    fn color_multiplier(self) -> (f32, f32, f32);
    fn particle_density(self) -> Option<(char, u32, Color)>; // (glyph, 1/N格)
}
```

## 二、潮湿系统 Wet

### 2.1 组件

```rust
#[derive(Debug, Clone, Copy)]
pub struct Wet {
    pub value: f32,  // 0=干爽, 100=浸透
}
```

出场时 `Wet { value: 0.0 }`。殖民者/俘虏/玩家各带一个，初始干爽。

### 2.2 累积速率

| 天气 | Δ/tick | 到100%的tick | 折合游戏时间 | 1x 真实秒 |
|------|:---:|:---:|:---|:---:|
| Drizzle | +0.15 | ~670 | ~80 min | ~67s |
| Rain | +0.4 | 250 | ~30 min | ~25s |
| Heavy | +1.0 | 100 | ~12 min | ~10s |
| Thunder | +1.5 | ~67 | ~8 min | ~7s |

Clear / Overcast 不增。

### 2.3 干燥速率

| 条件 | Δ/tick | 从100到0的tick |
|------|:---:|:---:|
| 室内（has_roof）| −0.12 | ~830 |
| 篝火邻格 dist≤2 | −0.8 | 125 |
| 手持火把 | −0.05 | ~2000 |
| 晴天户外 | −0.08 | ~1250 |

干燥条件可叠加（室内 + 烤火 = −0.92 / tick）。

### 2.4 分层效果

| 阈值 | 标签 | 心情 | 精力衰减 |
|:---:|------|:---:|:---:|
| 0-20 | 干爽的 | — | — |
| 21-50 | 潮湿的 | −3 | — |
| 51-80 | 湿透的 | −8 | +50% |
| 81-100 | 泡在水里 | −15 | +100% |

精力衰减加在 `needs.rs` 里：`energy_per *= (1.0 + wet_multiplier)`。

## 三、渲染

### 3.1 调色板乘数

```rust
fn weather_color_mult(weather: Weather) -> (f32, f32, f32) {
    match weather {
        Clear    => (1.00, 1.00, 1.00),
        Overcast => (0.88, 0.88, 0.82),
        Drizzle  => (0.85, 0.86, 0.82),
        Rain     => (0.78, 0.80, 0.76),
        Heavy    => (0.65, 0.68, 0.72),
        Thunder  => (0.55, 0.58, 0.65),
    }
}
```

现有 `map_view.rs` 已经在做日夜调色板：`(fg, bg) → 乘 night_mult`。天气乘数插在它前面或后面，最终：

```
color = original_color × night_mult × weather_mult
```

### 3.2 粒子叠加

在 `draw` 里画完地形/实体后，对每个视口格做：

```rust
if let Some((glyph, density, color)) = weather.particle_density() {
    if thread_rng().gen_range(0..density) == 0 {
        // 如果这格已经有实体字符 → 不覆盖（粒子在实体后面）
        // 如果这格是空地 → 画粒子
    }
}
```

粒子占的格必须是空地（没有实体 glyph），不然粒子穿模。有实体就跳过该格——粒子只在背景上飘。

### 3.3 闪电

Thunder 时每帧 `gen_bool(0.10..0.20)` 触发：

```
闪电帧：所有 Span 的 fg/bg 颜色亮度 × 1.5（clamp到1.0）；粒子全部白色
下一帧：恢复正常计算
```

实现方式：`App` 加 `lightning_flash: bool`，weather system 设置它，map_view 读它。

## 四、火源熄灭

户外火源（无 roof 的 LightSource 实体）每 tick 掷骰：

| 天气 | 概率/tick |
|------|:---:|
| Drizzle | 0 |
| Rain | 0.5% |
| Heavy | 3% |
| Thunder | 5% |

熄灭时：篝火 Entity despawn + mark_spatial_dirty + push_log "篝火被雨水浇灭了。" + emit `FireExtinguished` 事件。

## 五、视野折扣

`visibility_radius()` 返回值乘以天气乘数：

| 天气 | 乘数 |
|------|:---:|
| Clear / Overcast | 1.0 |
| Drizzle | 0.85 |
| Rain | 0.70 |
| Heavy | 0.50 |
| Thunder | 0.45 |

叠加在现有日夜视野外面：`final_radius = (night_radius as f32 * weather_mult) as i32`。
