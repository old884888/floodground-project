## Plan 00 — MVP：会呼吸、会记仇的一小片血壤

**目标**：验证「压榨 → 反噬」内核能跑通。

**技术选型**：Rust + hecs + ratatui + crossterm + serde/ron + rand。砍掉 bevy/egui/wgpu/noise/A*，追求秒级编译。

**核心交付**：
- 48×24 固定小地图，手搓地形
- 1 主角 + 2 殖民者 + 1 俘虏，需求系统（饥饿/精力/心情衰减）
- 殖民者极简 AI（吃/睡/闲逛优先级链）
- 刑讯 T 键 → 俘虏 will 下降 → 目击者心情暴跌 → 声誉下降
- L1 即时叙事日志（中文模板）
- ratatui 三区 UI（顶栏 + 地图面板 + 日志）

**关键决策**：
- 事件队列用 `Vec<GameEvent>`，system 按序调用普通函数。低耦合精神保留，实现从简。
- 数值先常量后 ron，别让数据管线卡迭代。
- 终端 RAII 守卫是硬性要求（panic 也要恢复终端）。

## Planmini1 — 日夜与大地：在黑暗中摸瞎砍树

**目标**：大地图走起来，天黑看不见，砍树挖矿摘莓果，两只手各拿一样东西。

**核心交付**：
- 500×500 地图，噪声生成水域/树木/岩石/灌木
- 视口跟随玩家 + 边缘 clamp
- 树/岩石/灌木升格为 Entity（Harvestable + HP + yield）
- 双手物品系统（Hands { left, right }），G 捡 C 砍 M 采 D 丢
- 日夜循环：一天 12000 tick，四时段（黎明/白天/黄昏/夜晚）
- CDDA 风格可见性：夜晚半径 8 格，篝火 LightSource 半径 15
- 调色板叠加（夜晚颜色阴沉）
- 区域刷新：热区 ≤30 / 温区 31-60 / 冷区 >60 冻结
- 动作锁定连砍/连采

**关键决策**：
- 视线阻挡用切比雪夫距离，暂不做 Bresenham。
- 需求衰减按天速率缩放（一天 12000 tick，别拿 MVP 的 120 tick 速率）。
- 白天视野后来收窄到 50（性能 + 游戏性）。

## Planmini2 — 狼 AI：追、逃、咬

**目标**：地图上有会追着你咬的东西。

**核心交付**：
- 全图 4 群狼，每群 1-3 只，离营 ≥30 格，群间 ≥40 格
- 行为树：追击（≤8 格 80%）/ 闲逛 / 逃跑
- 逃跑触发：独狼 HP≤60% 50% 概率；有同伴 HP≤40% 必逃
- 逃跑终止：距玩家 >15 格
- 咬人：dist≤1，冷却 1 tick，伤害 15~25

**关键决策**：
- 狼标记为 Hostile，selected 可控（primary_target 机制）。
- 玩家死后 primary_target 改追殖民者。

## Planmini3 — 翻垃圾：Pile 系统 + 两段式交互

**目标**：一格塞 128 种破烂不报错，"先看、再选、再拿"三段式交互。

**核心交付**：
- Pile 数据结构（Vec<PileSlot>，128 上限，同种堆叠）替代 GroundItem
- e 键两段式：按 e → 方向键选目标格 → 弹窗列出物品 → ↑↓选 G 捡
- 侧边栏三 Tab（[ ] 翻页）：角色 / 双手 / 营地
- Hands 升级为 `Option<(ItemKind, u32)>`，同种可叠
- F6 调试菜单（日夜快进迁入）
- inspect 模式下时间照走（CDDA 风格）

**关键决策**：
- Pile dominant() 每次遍历选最大 count——O(n) 可接受。
- 弹窗用 Clear widget 避免地图透底。

## Plan 04 — 村庄：CDDA 模板生成 + 室内黑暗

**目标**：地图上有能进能出的建筑，室内黑得伸手不见五指。

**核心交付**：
- CDDA palette ASCII 模板生成建筑（小屋/中屋/大屋/工坊/仓库）
- 模板旋转 4 朝向 + 15% 石墙材质替换
- 3 种村庄规模：小（3-5 围篝火）、中（5-8 沿街）、大（8-12 围栏+大门）
- 室内黑暗：roof 标记 + window_light 窗口导光（白天向内 3 格递减）
- 建筑组件：Wall/WoodWall/StoneWall/Door/Window/Bed/ContainerTag/Floor
- 交互：开门关门、床睡觉回精力、翻容器弹窗、砸墙掉落材料
- 观察系统 X 键：标记 focused_tile → 侧栏诙谐描述
- 持久迷雾三层状态（未探索/探索过但不在视野/当前可见）

**关键决策**：
- `has_roof` 存为 `GameMap.roof: Vec<bool>`，O(1) 查询。
- 窗导光预计算存入 `window_light` 表。
- 容器复用 Pile 组件 + Container marker。
- 村庄生成后 per-village rebuild 空间索引。

## Plan 05 — 制作系统 + 光照统一 + 主菜单

**目标**：旧石器工具链，光照五级统一计算，主菜单分流。

**难度**：S 级（8 分），跨 7+ 文件。

**核心交付**：

### 统一光照系统 `tile_light(x,y) → u8`
| 级别 | 名称 | 来源 | 制作倍率 |
|------|------|------|:---:|
| 0 | 完全看不见 | 夜晚户外无源、室内深处 | 不能做 |
| 1 | 昏暗 | 火把、篝火边缘、黄昏 | ×0.5 |
| 2 | 明亮 | 白天户外、篝火近旁 | ×1.0 |
| 3 | 太亮了 | 预留 | ×0.8 |
| 4 | 亮瞎眼 | 预留 | ×0 |

光源叠加：环境光(日夜) + 火源 LightSource { radius, brightness } + 手持火把(半径5) + 窗口光

### 制作系统
- R 键弹窗配方菜单 → ↑↓选 Enter 确认 Esc 关
- 自动收集 dist≤2 材料（双手 → 脚下 → dist=1 → dist=2）
- 进度条 tick + 旋转动画(⠋⠙⠹) + 百分比
- 光照不足暂停制作不取消；篝火邻格检查
- 5 条旧石器配方：石刀/削尖棍/火烤矛/石斧/火把

### 半成品系统（CraftWip）
- Esc 中断 → 生成 CraftWip 实体（黄色… glyph）→ 保留进度
- 站上去按 R → [续作] → 从上次进度继续，不扣材料
- progress=0 不生成 WIP；Recipe.can_interrupt=false 不可中断

### 主菜单
- Screen 枚举分流：MainMenu / Gameplay
- 启动 → 主菜单（开始/退出），↑↓ Enter 选择
- 游戏中 Q 直接退出到桌面
- 菜单状态不跑 tick

### 速度系统
- 五档：暂停 / 步进 / 正常 / 快进 / Turbo(15x)
- +/− 调速，制作菜单内也可调

### 其他改进
- 调试菜单重排：13 项，四段日夜合并为轮播
- 观察面板：Pile 全展开 + 生物伤势文字标签
- [ ] 键滚动侧边栏，移光标自动重置
- Pile 空壳自动清理

**技术债务（已知未修）**：
1. 火把作为实体 LightSource 而非双手物品检测
2. 制作菜单 Browsing 不暂停时间（CDDA 式设计）
3. `can_craft` 的 TOCTOU（同帧无并发，实际不触发）
4. 两个 WIP 同格时 wip_info_at 只返回第一个
