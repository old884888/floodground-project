## 零、设计摘要

当前 GameMap 是一整块 `Vec<Tile>`，没有区块概念。先切 Chunk，再挂脏标记，最后序列化。

**核心流程**：任何改变世界的操作 → 标记所在 Chunk dirty → 存档时只写 dirty Chunk → gzip 压缩 → 原子写入磁盘。

**读档流程**：解压 → 重建 GameMap（种子生成 + 读取 dirty Chunk 覆盖）→ spawn Entity（分配新 hecs Entity，建立 ID 映射）→ 还原全局状态。

## 一、Chunk 数据结构

### 1.1 Chunk 定义

```rust
pub const CHUNK_SIZE: i32 = 16;
// MAP_WIDTH/HEIGHT 不变，500×500 = 31×31 chunks + 边缘（4格）

#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct Chunk {
    pub cx: i32,           // chunk 坐标：cx = tile_x / 16
    pub cy: i32,
    pub tiles: Vec<Tile>,  // 256 个瓦片（16×16 展开）
    pub roof: Vec<bool>,
    pub window_light: Vec<u8>,
    pub revealed: Vec<bool>,
}

impl Chunk {
    pub fn new(cx: i32, cy: i32) -> Self {
        let size = (CHUNK_SIZE * CHUNK_SIZE) as usize;
        Self {
            cx, cy,
            tiles: Vec::with_capacity(size),
            roof: vec![false; size],
            window_light: vec![0; size],
            revealed: vec![false; size],
        }
    }
}
```

### 1.2 GameMap 重构

```rust
pub struct GameMap {
    pub width: i32,    // 500
    pub height: i32,   // 500
    chunks: HashMap<(i32, i32), Chunk>,  // 替代 Vec<Tile>
    dirty_chunks: HashSet<(i32, i32)>,   // 存档时只写这些
}
```

公开方法 `tile(x,y)`、`terrain(x,y)`、`is_walkable(x,y)` 等内部自动转换坐标到 Chunk 索引——外部调用不变。

### 1.3 坐标转换

```rust
pub fn chunk_coord(x: i32, y: i32) -> (i32, i32) {
    (x.div_euclid(CHUNK_SIZE), y.div_euclid(CHUNK_SIZE))
}

pub fn local_index(x: i32, y: i32) -> usize {
    let lx = x.rem_euclid(CHUNK_SIZE) as usize;
    let ly = y.rem_euclid(CHUNK_SIZE) as usize;
    ly * CHUNK_SIZE as usize + lx
}
```

### 1.4 脏标记

任何变动世界的方法调用 `mark_dirty(x, y)` → 该格所在 Chunk 加入 dirty_chunks。

| 操作 | 脏标记触发点 |
|------|------|
| 砍树（despawn Tree） | harvest.rs despawn 后 |
| 挖矿（despawn Boulder） | harvest.rs despawn 后 |
| 建造完成 | building.rs spawn 后 |
| 物品掉落/捡取 | items.rs place_item / take 后 |
| 陷阱触发 | building.rs despawn trap 后 |
| 地坑塌方 | weather.rs despawn PitShelter 后 |
| 灌木生长阶段变化 | nature.rs state 切换后 |
| 铺屋顶 | building.rs set_roof 后 |
| 开门/关门 | examine.rs toggle_door 后 |

所有以 `app.mark_spatial_dirty()` 结尾的地方都顺带调 `app.map.mark_dirty(x, y)`。

## 三、存档格式

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct SaveData {
    pub version: u32,          // 1
    pub tick: u64,
    pub day: u64,
    pub seed: u64,             // 世界种子
    pub weather: Weather,
    pub weather_timer: u64,
    pub reputation: i32,
    pub player_uid: u64,       // 当前玩家 EntityUID
    pub selected_uid: u64,     // 当前选中
    pub next_uid: u64,         // 下一次 spawn 用的 ID

    pub entities: Vec<EntitySnapshot>,
    pub dirty_chunks: Vec<Chunk>,   // 只有改过的
}
```

存档文件名：`saves/slot_01.ron.gz`

## 四、加载流程

```
1. 解压 gzip → RON 文本
2. 反序列化 → SaveData 结构体
3. 用 seed 生成一个新 GameMap（种子总是出同样的地形）
4. 遍历 dirty_chunks：逐个索引写入 tiles/roof/window/revealed
5. 遍历 entities：每个 EntitySnapshot → spawn Entity
   - 分配新 hecs Entity
   - HashMap<u64, Entity> 记录映射
   - 插 Component（用 Entity 映射处理 StickTrap.builder 这类跨实体引用）
6. 还原全局状态（tick/day/weather/reputation/player_uid/selected_uid）
7. 重建 SpatialIndex
8. App.world = new_world; App.map = new_map; 开始跑 tick
```
