## 零、设计摘要

补上生存闭环的两个缺口：**能喝水了**（Thirst 条终于有用了）、**有温度了**（天气不只是视觉效果）。

水喝法极其朴素——趴水边用嘴喝。脏水概率拉肚子。腹泻不是一次性惩罚，是持续半天到一天半的**效果**。效果系统按通用框架做，为以后流血/感染/中毒留位置。

温度走 CDDA 骨架：身体温度向环境温度缓慢趋近。无衣物/风速（v1 衣服就是没有）。

## 一、喝水系统

### 1.1 操作

走到水源格 → e 键 → 菜单出现"喝水"选项 → Enter → 喝水。

```
水源格判定: terrain == ShallowWater || Water || Stream || 水洼
```

### 1.2 效果

| 水源 | 口渴恢复 | 腹泻概率 | 日志 |
|------|:---:|:---:|------|
| 浅水(ShallowWater) | +30~40 | 8% | "你趴在水边猛灌了几口——有点腥，但管用。" |
| 深水(Water) | +30~40 | 3% | "湖心的水清澈些——也可能是错觉。" |
| 溪流(Stream) | +25~35 | 1% | "溪水流过石头，凉得牙根发酸。" |
| 雨后水洼 | +20~30 | 15% | "泥坑里的积水——你会后悔的。大概。" |

### 1.3 腹泻效果

```
6000-18000 tick（半天到一天半，约 10-30 现实分钟/正常速度）
口渴速度 ×1.5，心情 −10
日志开头: "肚子一阵绞痛——那水不对劲。"
日志结束: "肚子终于消停了。"
```

### 1.4 水洼（Puddle）

- 暴雨/雷阵雨后，浅沼地形上随机生成
- 单格实体，glyph `~` 泥黄色
- 持续时间 300-600 tick 后自然蒸发消失
- 玩家可以喝（但是最脏的）
- 生成：暴雨中每 tick 10% 概率在浅沼格生成（最多 20 个全图），雷阵雨 5%

## 二、溪流（Stream）

### 2.1 生成方式

不新增 TerrainKind。在噪声生成后处理：

```
找到相邻的浅水区之间最短路径 → 画 1-2 格宽 ShallowWater ≡ Stream
实际地形还是 ShallowWater，只是 diarrhea_chance 不同
```

或者更简单——加 `TerrainKind::Stream`：

```
Stream: 浅水变体, glyph="≈" 蓝色, is_walkable=true, move_cost=0.45, 
        比 ShallowWater 稍快（水流不黏脚），腹泻 1%
```

加一个新 TerrainKind 是十行代码的事——`components.rs` 一行 + `world.rs` 一行 + `terrain.ron` 一段 + `data.rs` 自动支持。

### 2.2 溪流数据

```
Stream: display_name="溪流", glyph="≈", fg(60,140,220), bg(0,15,40)
        is_walkable=true, move_cost=0.45, vis_mod=1.0, vis_flat=0
        rain_shield=0.0, auto_wet=true, diarrhea=1%
```

### 2.3 生成算法

噪声生成完浅水区后：随机选 3-5 对大浅水区（面积>20格）→ 每对之间用洪水填充画 1-2 格宽 Stream 连接。

## 三、温度系统（CDDA 骨架）

### 3.1 组件

```rust
#[derive(Debug, Clone, Copy)]
pub struct BodyTemp {
    pub value: f32,  // 0-100, 50=最舒适
}
```

开局所有角色 `BodyTemp { value: 60.0 }`。尸体没有体温。

### 3.2 环境温度公式

```
环境温度 = 基础(50) + 天气 + 湿身惩罚 + 火源 + 室内

天气基准:
  晴天白天  +20    晴天夜晚  -8
  阴天白天  +10    阴天夜晚  -12
  毛毛雨白天 +5    毛毛雨夜晚 -10
  雨白天    -5     雨夜晚    -15
  暴雨白天  -10    暴雨夜晚  -18
  雷阵雨白天 -8    雷阵雨夜晚 -18

湿身惩罚(Wet):
  Wet ≤ 50: 0
  Wet 51-80: -5
  Wet > 80: -10

火源:
  篝火(≤2格): +25
  手持火把: +8

室内(has_roof): +10
```

### 3.3 体温趋近

```rust
// needs.rs 每 tick:
let diff = env_temp - body_temp.value;
body_temp.value += diff * 0.3; // 每 tick 向环境温度移 30% 差距
body_temp.value = body_temp.value.clamp(0.0, 100.0);
```

### 3.4 温度分层效果

| BodyTemp | 标签 | 效果 |
|:---:|------|------|
| 60-100 | 热的 | Thirst 衰减 +50% |
| 45-60 | 舒服 | 正常 |
| 30-45 | 冷 | 移速 -10%，制作速度 -20%，精力衰减 +30% |
| 15-30 | 很冷 | 移速 -25%，制作 -50%，精力衰减 +50% |
| 0-15 | 失温 | 每 50 tick 扣 1-3 HP，X 观察标签"在发抖" |
| 0 | 冻死 | 触发死亡（kill，"冻死了"） |

## 四、效果系统（StatusEffect）

### 4.1 组件

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum EffectKind {
    Diarrhea,
    // 预留
    // Bleeding,
    // Infection,
    // Poison,
}

#[derive(Debug, Clone)]
pub struct StatusEffect {
    pub kind: EffectKind,
    pub remaining: u32, // tick 数
}
```

一个实体可以有多个 StatusEffect。通过 hecs 的 `Vec<StatusEffect>` 组件。

### 4.2 效果 tick

```rust
// needs.rs 里，处理完基础需求衰减后：
for (e, effects) in app.world.query::<&mut Vec<StatusEffect>>().iter() {
    effects.retain_mut(|eff| {
        apply_effect(eff, app, e);
        eff.remaining = eff.remaining.saturating_sub(1);
        eff.remaining > 0
    });
}
```

### 4.3 Diarrhea 具体效果

```rust
fn apply_effect(eff: &StatusEffect, app: &mut App, entity: Entity) {
    match eff.kind {
        EffectKind::Diarrhea => {
            // 渴得更快：在 needs.rs 的 thirst 衰减处额外乘 1.5
            // 心情惩罚：在 mood 计算处 -10
            // （这些在 needs.rs 主循环里读取 effects 列表实现）
        }
    }
}
```

实际实现：在 `update_needs` 里，算完基础衰减后，检查实体是否有 Diarrhea 效果 → 如果有，`thirst_rate *= 1.5`，`mood -= 10.0 / ticks_per_day`（封顶）。

效果结束时刷日志"肚子终于消停了。"
